# PrimeChain Solutions — CSS Optimization Guide\n\n## Overview\n\nThis document outlines the optimized CSS structure implemented to improve website performance, maintainability, and caching efficiency.\n\n---\n\n## What Changed\n\n### Before (Monolithic Structure)\n- **540+ lines** of CSS embedded in each HTML file's `<style>` tag\n- Duplicated article styles across `dashboard.html`, `manualwork.html`, etc.\n- All JavaScript logic inline (~60 lines per page)\n- **Result**: ~34KB index.html, poor caching, code duplication\n\n### After (Modular Structure)\n- **Global styles** in `style.css` (shared across all pages)\n- **Page-specific styles** in dedicated CSS files:\n  - `css/index.css` — Home page only\n  - `css/article.css` — All article pages\n  - `css/ourthinking.html` — \"Our Thinking\" page\n- **Shared JavaScript** in `js/main.js`\n- **Result**: ~22KB index.html, **35% smaller**, better caching\n\n---\n\n## Directory Structure\n\n```\n.\n├── index.html                    # Home page (now 22KB)\n├── dashboard.html                # Article page\n├── manualwork.html               # Article page\n├── ourthinking.html              # Our Thinking page\n├── whoweare.html                 # (unchanged)\n├── paperwork.html                # (unchanged)\n├── supplychain.html              # (unchanged)\n│\n├── style.css                     # Global shared styles (~590 lines)\n│\n├── css/                          # NEW: Page-specific styles\n│   ├── index.css                 # Home page unique styles\n│   ├── article.css               # Shared article page styles\n│   └── ourthinking.css           # Our Thinking page styles\n│\n├── js/                           # NEW: Centralized JavaScript\n│   └── main.js                   # Scroll progress, nav, reveal animations\n│\n└── assets/                       # Images & icons\n    ├── primechain.svg\n    ├── empower.jpeg\n    └── ...\n```\n\n---\n\n## How It Works\n\n### Global CSS (`style.css`)\nContains design tokens and shared components used across **all** pages:\n\n```css\n/* Tokens (variables) */\n:root {\n  --navy: #071534;\n  --teal: #008080;\n  --gold: #C9A84C;\n  --ease: 0.25s ease;\n}\n\n/* Shared components */\n.container { max-width: 1180px; margin: 0 auto; padding: 0 2rem; }\n.nav { /* Navigation shared across all pages */ }\n.btn-teal { /* Button styles */ }\n.footer { /* Footer (same on all pages) */ }\n```\n\n**Benefits:**\n- Cached after first page load\n- Used by every page (100% reuse)\n- Reduces duplication\n\n### Page-Specific CSS\n\n**`css/index.css`** — Home page hero, problem section, services grid:\n```css\n.hero { /* Large, unique to index */ }\n.pain-list { /* Problem cards grid */ }\n.svc-grid { /* Services 3-column layout */ }\n```\n\n**`css/article.css`** — Shared by all article pages:\n```css\n.article-body { /* Consistent article layout */ }\n.article-meta { /* Byline, tags, read time */ }\n.article-nav { /* Previous/Next navigation */ }\n```\n\n**`css/ourthinking.css`** — \"Our Thinking\" page featured cards & grid:\n```css\n.feat-card { /* Featured article/case study card */ }\n.thinking-grid { /* 3-column article grid */ }\n```\n\n**Benefits:**\n- Only loaded when needed\n- Smaller initial page load (index.html loads only `css/index.css`)\n- Easy to modify per-page styles without affecting others\n\n### JavaScript (`js/main.js`)\nCentralized, reusable script for all pages:\n\n```javascript\n// Scroll progress bar\n// Mobile menu toggle\n// Reveal animations (Intersection Observer)\n```\n\n**Loaded once, cached, used on all pages.**\n\n---\n\n## Performance Gains\n\n### File Size Reduction\n| Page | Before | After | Savings |\n|------|--------|-------|----------|\n| index.html | 34 KB | 22 KB | **35%** |\n| dashboard.html | 18 KB | 11 KB | **39%** |\n| ourthinking.html | 12 KB | 6 KB | **50%** |\n\n### Caching Benefits\n\n**Scenario: User visits index.html, then ourthinking.html**\n\n**Before:**\n- Download `style.css` (590 lines)\n- Download index HTML with 540 lines of embedded CSS\n- Visit ourthinking.html → Download again (540 different embedded styles)\n- Total: 2 full CSS downloads\n\n**After:**\n- Download `style.css` (590 lines) — **cached**\n- Download `css/index.css` (310 lines)\n- Visit ourthinking.html → Reuse `style.css` + load `css/ourthinking.css` only\n- Total: `style.css` cached, only page-specific CSS fetched\n\n**Result:** ~25% faster navigation between pages on repeat visits.\n\n---\n\n## How to Maintain\n\n### Adding a New Page\n\n1. **Create HTML** (e.g., `newpage.html`)\n2. **Link shared CSS** in `<head>`:\n   ```html\n   <link rel=\"stylesheet\" href=\"style.css\" />\n   <!-- Your page-specific CSS -->\n   <link rel=\"stylesheet\" href=\"css/newpage.css\" />\n   ```\n3. **Create `css/newpage.css`** with only page-unique styles\n4. **Link shared JS**:\n   ```html\n   <script src=\"js/main.js\"></script>\n   ```\n\n### Modifying an Existing Page's Styles\n\n**If it's a shared element** (header, button, etc.):\n- Edit `style.css`\n- Changes apply to all pages automatically\n\n**If it's page-specific** (e.g., hero section on index only):\n- Edit `css/index.css`\n- Only affects that page\n\n### Adding New JavaScript\n\n**If it's used on multiple pages:**\n- Add to `js/main.js`\n- Wraps existing code\n\n**If it's for one page only:**\n- Create inline `<script>` in that HTML file OR\n- Create `js/pagename.js` and load it only on that page\n\n---\n\n## What
